SKEMA DATABASE - SISTEM MANAJEMEN SDM (v3.0 - SQL)
================================================

PENJELASAN
----------
Dokumen ini merinci skema database SQL (SQLite) yang digunakan oleh aplikasi.
Struktur ini lebih formal dan kuat dibandingkan dengan versi berbasis file-JSON sebelumnya.

- Tipe Data: Menggunakan tipe data afinitas SQLite (TEXT, INTEGER, REAL).
- Kunci: Menandai Primary Key (PK) untuk setiap tabel.
- Penyimpanan JSON: Kolom yang menyimpan data terstruktur kompleks (seperti riwayat pekerjaan)
  menggunakan tipe data `TEXT` untuk menyimpan string JSON. Proses serialisasi (stringify)
  dan deserialisasi (parse) ditangani di backend (`server.js`).

---

### 1. Tabel: `users`
Menyimpan data akun pengguna untuk login, otentikasi, dan manajemen akses.

| Nama Kolom          | Tipe Data         | Deskripsi                                                        |
|---------------------|-------------------|------------------------------------------------------------------|
| id                  | TEXT PRIMARY KEY  | ID unik untuk setiap pengguna, cth: 'user-001'.                  |
| name                | TEXT              | Nama lengkap pengguna.                                           |
| email               | TEXT UNIQUE       | Alamat email unik untuk login.                                   |
| role                | TEXT              | Peran pengguna: 'ADMIN' atau 'EMPLOYEE'.                         |
| employeeId          | TEXT              | Foreign Key yang merujuk ke `employees.id`.                      |

---

### 2. Tabel: `employees`
Menyimpan data detail dan komprehensif untuk setiap karyawan.

| Nama Kolom             | Tipe Data         | Deskripsi                                                        |
|------------------------|-------------------|------------------------------------------------------------------|
| id                     | TEXT PRIMARY KEY  | ID unik untuk setiap karyawan, cth: 'emp-001'.                   |
| nip                    | TEXT UNIQUE       | Nomor Induk Pegawai.                                             |
| position               | TEXT              | Jabatan karyawan, cth: 'Pengembang Frontend'.                    |
| pangkat                | TEXT              | Nama pangkat atau tingkatan karyawan.                            |
| golongan               | TEXT              | Kode golongan ruang, cth: 'III/a'.                               |
| department             | TEXT              | Departemen tempat karyawan bekerja.                              |
| joinDate               | TEXT              | Tanggal karyawan mulai bergabung (YYYY-MM-DD).                   |
| avatarUrl              | TEXT              | URL ke foto profil karyawan.                                     |
| leaveBalance           | INTEGER           | Sisa kuota cuti tahunan (dalam hari).                            |
| isActive               | INTEGER           | Status keaktifan (1 = aktif, 0 = nonaktif).                      |
| address                | TEXT              | Alamat tempat tinggal.                                           |
| phone                  | TEXT              | Nomor telepon aktif.                                             |
| pob                    | TEXT              | Tempat lahir.                                                    |
| dob                    | TEXT              | Tanggal lahir (YYYY-MM-DD).                                      |
| religion               | TEXT              | Agama yang dianut.                                               |
| maritalStatus          | TEXT              | 'Lajang', 'Menikah', 'Bercerai', 'Janda/Duda'.                   |
| numberOfChildren       | INTEGER           | Jumlah anak.                                                     |
| educationHistory       | TEXT              | String JSON dari array objek riwayat pendidikan.                 |
| workHistory            | TEXT              | String JSON dari array objek riwayat pekerjaan.                  |
| trainingCertificates   | TEXT              | String JSON dari array objek sertifikat.                         |
| payrollInfo            | TEXT              | String JSON dari objek pengaturan penggajian.                    |

---

### 3. Tabel: `leaveRequests`
Mencatat semua pengajuan cuti atau izin dari karyawan.

| Nama Kolom             | Tipe Data         | Deskripsi                                                        |
|------------------------|-------------------|------------------------------------------------------------------|
| id                     | TEXT PRIMARY KEY  | ID unik untuk setiap pengajuan.                                  |
| employeeId             | TEXT              | Foreign Key ke `employees.id`.                                   |
| employeeName           | TEXT              | Nama karyawan (denormalisasi).                                   |
| leaveType              | TEXT              | 'Cuti Tahunan', 'Cuti Sakit', 'Izin Khusus'.                     |
| startDate              | TEXT              | Tanggal mulai cuti (YYYY-MM-DD).                                 |
| endDate                | TEXT              | Tanggal selesai cuti (YYYY-MM-DD).                               |
| reason                 | TEXT              | Alasan pengajuan cuti.                                           |
| status                 | TEXT              | 'Menunggu', 'Disetujui', 'Ditolak'.                              |
| supportingDocument     | TEXT              | Nama file atau path ke dokumen pendukung.                        |
| rejectionReason        | TEXT              | Alasan jika pengajuan ditolak.                                   |

---

### 4. Tabel: `payrolls`
Menyimpan riwayat slip gaji yang telah dibuat untuk setiap karyawan per periode.

| Nama Kolom             | Tipe Data         | Deskripsi                                                        |
|------------------------|-------------------|------------------------------------------------------------------|
| id                     | TEXT PRIMARY KEY  | ID unik untuk setiap slip gaji.                                  |
| employeeId             | TEXT              | Foreign Key ke `employees.id`.                                   |
| employeeName           | TEXT              | Nama karyawan (denormalisasi).                                   |
| period                 | TEXT              | Periode penggajian, cth: 'Juni 2024'.                            |
| baseSalary             | REAL              | Gaji pokok pada periode tersebut.                                |
| incomes                | TEXT              | String JSON dari array komponen pendapatan.                      |
| deductions             | TEXT              | String JSON dari array komponen potongan.                        |
| totalIncome            | REAL              | Jumlah total pendapatan.                                         |
| totalDeductions        | REAL              | Jumlah total semua potongan.                                     |
| netSalary              | REAL              | Gaji bersih (take home pay).                                     |

---

### 5. Tabel: `performanceReviews`
Menyimpan data hasil penilaian kinerja karyawan.

| Nama Kolom             | Tipe Data         | Deskripsi                                                        |
|------------------------|-------------------|------------------------------------------------------------------|
| id                     | TEXT PRIMARY KEY  | ID unik untuk setiap penilaian.                                  |
| employeeId             | TEXT              | Foreign Key ke `employees.id`.                                   |
| employeeName           | TEXT              | Nama karyawan (denormalisasi).                                   |
| period                 | TEXT              | Periode penilaian, cth: 'Q2 2024'.                               |
| reviewerName           | TEXT              | Nama penilai (denormalisasi).                                    |
| reviewDate             | TEXT              | Tanggal penilaian (YYYY-MM-DD).                                  |
| overallScore           | REAL              | Skor akhir keseluruhan dari penilaian.                           |
| status                 | TEXT              | 'Completed', 'In Progress'.                                      |
| strengths              | TEXT              | Umpan balik kualitatif mengenai kekuatan karyawan.               |
| areasForImprovement    | TEXT              | Umpan balik mengenai area yang perlu ditingkatkan.               |
| employeeFeedback       | TEXT              | Tanggapan dari karyawan terkait hasil penilaian.                 |
| kpis                   | TEXT              | String JSON dari array rincian KPI.                              |

---

### 6. Tabel: `attendance`
Mencatat data kehadiran harian setiap karyawan.

| Nama Kolom             | Tipe Data         | Deskripsi                                                        |
|------------------------|-------------------|------------------------------------------------------------------|
| id                     | TEXT PRIMARY KEY  | ID unik.                                                         |
| employeeId             | TEXT              | Foreign Key ke `employees.id`.                                   |
| employeeName           | TEXT              | Nama karyawan (denormalisasi).                                   |
| date                   | TEXT              | Tanggal absensi (YYYY-MM-DD).                                    |
| clockIn                | TEXT              | Waktu masuk (HH:mm:ss).                                          |
| clockOut               | TEXT              | Waktu keluar (HH:mm:ss).                                         |
| status                 | TEXT              | 'Tepat Waktu', 'Terlambat', 'Absen'.                             |
| workDuration           | TEXT              | Total durasi kerja, cth: '8j 15m'.                               |
